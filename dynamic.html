Memoization, tabulation, overlapping subproblems

Overview

Dynamic programming (DP) converts exponential recursive solutions into polynomial time using memoization or bottom-up tabulation. Two main properties: optimal substructure and overlapping subproblems.

Patterns

  • 1D DP — linear sequence (e.g., Fibonacci, climbing stairs)
  • 2D DP — grid/substring (e.g., edit distance, longest common subsequence)
  • Knapsack / DP over subsets

Example — Fibonacci

Memoization (JS)

function fib(n, memo = {}) {
  if (n <= 1) return n;
  if (memo[n]) return memo[n];
  memo[n] = fib(n-1,memo) + fib(n-2,memo);
  return memo[n];
}

Exercises

  1. Climbing stairs: number of ways to reach nth stair (1 or 2 steps at a time).
  2. Longest Increasing Subsequence (LIS) — O(n log n) solution (bonus) and O(n²) DP.
  3. 0/1 Knapsack: maximize value given weight capacity.

Solutions (hints)

  • Climb: fib-like DP or iterative loop storing dp[i] = dp[i-1] + dp[i-2].
  • LIS: patience sorting / binary search approach gives O(n log n).
  • Knapsack: classic DP with dp[w] or dp[i][w] depending on approach.
